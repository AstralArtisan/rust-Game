# 勇闯方块城 — 全面重构设计文档

## 1. 背景与目标

原项目 `E:/rust_game_merge` 经过多轮迭代，代码架构混乱、系统耦合严重、多处 bug 难以修复。本次重构在 `E:/block city` 全新重写整个项目，保留核心玩法设计，彻底重建代码架构。

目标：
- 清晰的模块化架构，每个系统职责单一、接口明确
- 数据驱动，所有数值外置到 RON 配置
- 强化系统从 2 层扩展到 3 层，第 3 层质变
- 修复事件房、奖励房、商店房的逻辑 bug
- 重写多人模式，与单机共用游戏逻辑
- 像素风美术统一，UI 精细化
- 为 NG+ 和后续扩展预留空间

## 2. 技术栈

### 3.3 伤害管线

独立 System 链，通过 Event 传递：

```
碰撞检测系统 → DamageEvent
  → immunity_check_system（无敌帧、幽灵态）→ 过滤
  → shield_check_system（冲刺护盾、词缀护盾）→ 吸收或通过
  → damage_modifier_system（破甲、方向防御、Boss阶段倍率）→ 修正数值
  → apply_damage_system → 扣血 → DamageAppliedEvent
  → post_damage_system（击退、吸血、连锁闪电、死亡判定、伤害数字）
```

Boss 特殊逻辑通过组件标记（BossPhase, DirectionalDefense）参与修正阶段，不硬编码 if-else。

## 4. 强化系统（Augment）

### 4.1 结构

- 固定 3 层：基础 → 进阶 → 完美
- 第 3 层产生质变效果
- 4 类别：近战(Melee)、远程(Ranged)、机动(Mobility)、通用(General)
- 3 稀有度：普通、精英、传说

### 4.2 获取途径

- 战斗房通关：必定掉落 1 个强化选择（3 选 1）
- 精英房通关：掉落精英/传说级强化选择
- 奖励房"强化锻造"：升级已有强化
- 商店：购买强化或强化升级服务
- 事件房：特定事件可获得强化
- Boss 宝箱：精英/传说强化（3 选 1）

### 4.3 升级途径

- 再次获得同一强化时自动升级
- 奖励房"强化锻造"选择升级
- 商店购买"强化升级服务"（120 金币）

## 5. 能量与终结技系统

### 5.1 能量机制

- 上限默认 100，可通过升级/商店/强化提升
- 获取：近战命中 +2、远程命中 +1、击杀 +5、拾取能量球 +15
- 不自然回复，鼓励主动战斗

### 5.2 终结技槽位

- 4 个槽位，按数字键 1-4 释放
- 解锁：1 号初始，2 号第 2 层，3 号第 3 层，4 号第 4 层
- 获取：Boss 宝箱必给终结技选择，奖励房/商店也可获得
- 替换：获得新终结技时选择装入哪个槽位

### 5.3 终结技消耗与 CD

| 档位 | 能量消耗 | CD | 终结技 |
|------|----------|-----|--------|
| 轻量级 | 60 | 8s | 战吼、弹幕倾泻、裂地斩 |
| 中量级 | 80 | 15s | 剑舞、冰晶领域、生命汲取、处刑之刃 |
| 重量级 | 全部能量(≥80) | 25s | 天降陨石、时空裂隙 |

## 8. 地图与房间流程

### 8.1 每层结构（10 房间）

```
房间 1：固定战斗房
房间 2-9：每次 2-3 个门，玩家选择
房间 10：固定 Boss 房
```

门生成规则：
- 必须包含至少 1 个战斗房选项
- 奖励房每层最多 1 次
- 商店/事件不连续出现
- 精英房从第 3 房间起可出现
- 只有一种合理选择时只生成 1 个门

每层预期：战斗 3-5 次、事件 1-2 次、商店 1-2 次、奖励 0-1 次、精英 0-1 次。

### 8.4 存档系统

- 读档从房间初始状态加载（战斗中存档退出后重进，房间重置）
- 存档内容：楼层、房间进度、玩家属性、强化背包、终结技槽位、金币、已访问房间、NG+ 状态

### 8.5 NG+（通关后）

每次通关解锁一个永久能力：

| 通关次数 | 解锁 |
|----------|------|
| 1 次 | "传承"：新游戏可携带上局 1 个普通强化(Lv1) |
| 2 次 | "先知之眼"：门上显示怪物数量和是否有精英 |
| 3 次 | "冒险者背包"：开局自带 1 治疗药水 + 1 能量药水 |
| 4 次 | "命运改写"：每层可免费重选一次门 |
| 5 次 | "大师起手"：开局从 3 个终结技中选 1 装备 |

难度选择：
- 普通：默认
- 困难（3 次通关解锁）：敌人 HP ×1.3，伤害 ×1.2，掉落 ×0.85
- 噩梦（5 次通关解锁）：敌人 HP ×1.6，伤害 ×1.4，掉落 ×0.7，精英必定双词缀，Boss 新增阶段

## 9. 多人模式

### 9.1 合作模式（Lightyear 0.26，主机权威）

模块结构：
- `coop/protocol.rs`：复制组件、消息、通道定义
- `coop/lobby.rs`：大厅连接、匹配、准备
- `coop/sync.rs`：实体/输入/状态同步
- `coop/authority.rs`：主机权威逻辑
- `coop/visuals.rs`：远程玩家视觉
- `coop/scaling.rs`：合作数值缩放
- `coop/ui.rs`：合作专用 UI

核心原则：
- 单机和合作共用同一套游戏逻辑，通过 `SessionMode` 资源区分
- 不在游戏逻辑中散布 if coop 分支，差异通过网络同步层处理
- 合作特殊逻辑封装在 coop 模块内

关键设计：
- 掉落物：两人独立拾取同一掉落物，各自获得
- 门选择：各自投票，相同直接进入，不同随机选一个
- 事件房：合作模式保留事件房
- 强化选择：各自独立
- 死亡：变幽灵态，通关当前房间后自动复活 50% HP
- 数值缩放：怪物 HP ×1.6，伤害 ×1.2，数量 +30%

### 9.2 PVP 模式（自写 UDP，轻量原型）

模块结构：
- `pvp/protocol.rs`：UDP 消息定义
- `pvp/net.rs`：网络收发
- `pvp/arena.rs`：竞技场逻辑
- `pvp/visuals.rs`：对手视觉
- `pvp/ui.rs`：PVP 专用 UI

原型功能：两人固定竞技场，近战+远程+冲刺对战，先死者输。预留扩展空间（道具、地图选择、多回合）。
